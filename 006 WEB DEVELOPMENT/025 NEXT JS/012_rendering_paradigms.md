✅ What is "use client"?

"use client" tells Next.js:

This component should run in the browser (client-side)

By default, Next.js uses Server Components.
You must add "use client" only when needed.


When Do You NEED "use client"?

You need "use client" if you use:
	•	useState
	•	useEffect
	•	useContext
	•	Event handlers (onClick, onChange)
	•	Browser APIs (window, document)

```js
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </>
  );
}
```

- By default, components in Next.js are Server Components
- Writing `"use client"` at the top makes the component a Client Component
- All child components of a Client Component also become Client Components
- `"use client"` works downward, not upward
- Use `"use client"` only when you need state, effects, or events
- Avoid adding `"use client"` unnecessarily to improve performance




## Advantages of "use client"

- Enables interactivity (clicks, forms, animations)
- Allows use of React hooks like useState and useEffect
- Access to browser APIs (window, document, localStorage)
- Required for event handlers (onClick, onChange, etc.)
- Good for UI-focused components (modals, dropdowns, counters)

## Disadvantages of "use client"

- Runs only in the browser (not on server)
- Less SEO friendly compared to Server Components
- Larger JavaScript bundle sent to client
- Slower initial page load
- All child components also become client components
- Overuse can hurt performance and scalability
